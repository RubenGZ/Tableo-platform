claim_slot — implementación completa (migración 006)
CREATE OR REPLACE FUNCTION claim_slot(
  p_resource_id  uuid,
  p_start_at     timestamptz,
  p_end_at       timestamptz,
  p_session_id   text,
  p_ttl_minutes  int DEFAULT 5
)
RETURNS jsonb
LANGUAGE plpgsql SECURITY DEFINER
AS $$
DECLARE
  v_id uuid;
BEGIN
  -- Conflict check (overlap formula: a.start < b.end AND a.end > b.start)
  IF EXISTS (
    SELECT 1 FROM bookings
    WHERE resource_id = p_resource_id
      AND status IN ('reserved', 'confirmed')
      AND start_at < p_end_at
      AND end_at   > p_start_at
  ) THEN
    RETURN jsonb_build_object('success', false, 'reason', 'not_available');
  END IF;

  INSERT INTO bookings (resource_id, start_at, end_at, status, reserved_until, session_id, metadata)
  VALUES (p_resource_id, p_start_at, p_end_at, 'reserved',
          NOW() + (p_ttl_minutes || ' minutes')::interval,
          p_session_id, '{}'::jsonb)
  RETURNING id INTO v_id;

  RETURN jsonb_build_object(
    'success',        true,
    'booking_id',     v_id,
    'reserved_until', NOW() + (p_ttl_minutes || ' minutes')::interval
  );

EXCEPTION
  WHEN unique_violation THEN
    RETURN jsonb_build_object('success', false, 'reason', 'concurrent_lock');
END;
$$;

7b — BeautyAdapter.getSlots() — Algoritmo paso a paso
1
Calcular bounds del día en UTC
Llamar dayBoundsUTC(date, timezoneId) → obtener startOfDay y endOfDay como objetos Date en UTC.
2
Convertir day_of_week
JS getDay(): 0=Dom, 1=Lun, …, 6=Sáb. DB: 0=Lun, 6=Dom.
Conversión: dbDayOfWeek = jsDay === 0 ? 6 : jsDay - 1
3
Query availability_windows
WHERE resource_id = ? AND day_of_week = dbDayOfWeek. Si no hay ventanas → return [].
4
Leer duración del recurso
resources.metadata.duration_default (minutos). Default: 60 min si no está definido.
5
Query blocking_rules
WHERE resource_id = ? AND start_at < endOfDay AND end_at > startOfDay
6
Query bookings ocupadas
WHERE resource_id = ? AND status IN ('reserved','confirmed') AND overlap con el día.
7
Generar slots por ventana
Para cada window: construir openUTC/closeUTC con new TZDate(year, month-1, day, H, M, 0, tz). Iterar cursor += durationMs mientras cursor + durationMs <= closeUTC.
8
Filtrar slots ocupados
Overlap formula: a.start < b.end && a.end > b.start. Aplica sobre blocks + bookings combinados.
9
Return Slot[]
Todas las fechas en UTC. La capa de presentación usa formatSlotTime() para mostrar hora local.

8Patrones Críticos

Supabase Mock en Vitest — patrón correcto
Advertencia: mock incorrecto causa hangs infinitos en tests
Usar vi.fn().mockResolvedValue() en el campo then de la chain de Supabase rompe el patrón thenable y hace que los tests con await no resuelvan nunca.
CORRECTO — thenable real
// ✅ Thenable real — resuelve correctamente
chain.then = (fn) =>
  Promise.resolve({ data, error: null }).then(fn)

// Así se construye un mock de query chain:
const chain = {
  select: vi.fn().mockReturnThis(),
  eq:     vi.fn().mockReturnThis(),
  single: vi.fn().mockReturnThis(),
  then: (fn) =>
    Promise.resolve({ data: mockData, error: null }).then(fn),
}
INCORRECTO — causa hangs
// ❌ NUNCA hacer esto:
chain.then = vi.fn().mockResolvedValue({
  data: mockData, error: null
})

// vi.fn() no implementa correctamente
// el protocolo Promise/A+.
// El await nunca resuelve → timeout
// del test runner.
TZDate Constructor — ADR-006
CORRECTO — parts constructor
// ✅ Interpreta como hora local del timezone
new TZDate(
  year, month - 1, day,
  hour, min, 0,
  timezoneId
)

// Ejemplo: 9:00 hora Madrid
// → la hora UTC depende del DST
new TZDate(2024, 5, 17, 9, 0, 0,
  'Europe/Madrid'
)
INCORRECTO — string constructor
// ❌ Interpreta como hora de la MÁQUINA,
// no del timezone indicado
new TZDate(
  '2024-06-17T09:00:00',
  'Europe/Madrid'
)

// En un servidor en UTC, '09:00:00' se
// interpreta como 09:00 UTC, no 09:00 Madrid
// → offset incorrecto en producción
Regla de Timezones — ADR-006
DB / Almacenamiento
Siempre UTC (timestamptz). Sin excepciones.
Capa de negocio / Módulos
Operan en UTC. Nunca convierten a zona local.
Presentación (UI)
Única capa que llama a dates.ts para mostrar hora local.
